Do & Don't

Non-negotiable rules for maintaining visual consistency across the team. Share with all four developers before implementation begins.

✓ Do
  • Always use --color-brand-primary for the single primary CTA per screen
  • Use --gold-400 exclusively for star ratings, pricing figures, and the "Confirm & Pay" button
  • Use Maroon only for Qatar National Day, Ramadan, and formal vendor onboarding — never as a regular UI color
  • Apply --shadow-brand to all primary teal buttons
  • Maintain --shadow-focus ring on all interactive elements for accessibility
  • Use Sora for all headings, buttons, labels, prices. Use DM Sans for all body copy
  • Use Noto Kufi Arabic for all Arabic text — never Sora or DM Sans in RTL mode
  • Always mirror layouts horizontally when switching to Arabic (RTL)
  • Only reference semantic tokens in components (never raw hex values)
  • Use Pearl 900 (#1E2025) for all primary text — never pure #000000
✕ Don't
  • Never use Maroon as a primary color in everyday UI — it is a heritage accent only
  • Never place two primary CTA buttons side by side on the same screen
  • Never use raw hex values in component code — always use CSS variables
  • Never use Gold as a background color for large surfaces — only for accents and icons
  • Never display Arabic text using Sora or DM Sans — always switch to Noto Kufi Arabic
  • Never use pure black (#000000) or pure white (#FFFFFF) for text — use Pearl scale
  • Never disable the focus ring for accessibility reasons
  • Never use the shadow-2xl for anything other than full-screen overlays/drawers
  • Never add a new color outside this system without design review
  • Never use Inter, Roboto, or system fonts — always load Sora and DM Sans from CDN

Dark Mode — Token System

Dark mode is not a color inversion. Every token is independently recalibrated for dark surfaces, taking into account Qatar's indoor/nighttime usage context, OLED screen physics, and the Gulf Teal system's behavior under dark conditions. Use the toggle button (bottom-right) to switch modes live.

Why these dark values — the reasoning
🌙
Qatar Night Context
A significant portion of car wash bookings happen in the evening — after work, during Ramadan nights, or in indoor malls. Dark mode isn't a trend feature here; it's a usage-context response. Deep surface backgrounds reduce eye strain during evening booking flows.
📱
OLED-First Dark Surfaces
Qatar's dominant devices (Samsung Galaxy S-series, iPhone Pro) use OLED panels where true-dark pixels consume zero power. Our dark backgrounds sit at #0C0F12 — dark enough for OLED savings, light enough to avoid halation around bright UI elements.
🌊
Teal Shifts One Step Brighter
On dark backgrounds, teal-500 (#0B8FBF) loses some contrast punch. We shift the primary to teal-400 (#0FA8D4) — one step brighter — to maintain the 4.5:1 WCAG AA minimum against dark surfaces without changing brand identity.
✨
Gold Stays Unchanged
Desert Gold #DEB84D is the one color that naturally elevates on dark backgrounds — it glows. No adjustment needed. This is actually where gold performs best, making ratings and pricing feel premium in dark mode without any modification.
🇶🇦
Maroon Finally Shines in Dark
Qatar Maroon actually performs better as a heritage accent in dark mode. We shift to maroon-400 (#C24E68) — a lighter, richer step — so National Day campaign screens feel vibrant and bold against deep backgrounds, not muddy.
🎨
Layered Dark Surfaces, Not Flat
Three distinct dark background levels: page canvas #0C0F12, card surface #141820, raised card #1C2029. This creates the same elevation hierarchy as light mode but using luminosity instead of shadows.
Live Preview — Light vs Dark
Both screens show identical content. This is exactly how your Flutter and Angular apps will render. Toggle using the 🌙 button bottom-right.

// ── Angular: Theme toggle service ────────────────
// lib/services/theme.service.ts

import { Injectable, signal } from '@angular/core';

@Injectable({ providedIn: 'root' })
export class ThemeService {
  private readonly STORAGE_KEY = 'cws-theme';
  isDark = signal(false);

  init(): void {
    // 1. Check saved preference
    const saved = localStorage.getItem(this.STORAGE_KEY);
    // 2. Fallback to system preference
    const prefersDark = window.matchMedia(
      '(prefers-color-scheme: dark)'
    ).matches;
    const dark = saved ? saved === 'dark' : prefersDark;
    this.apply(dark);
  }

  toggle(): void {
    this.apply(!this.isDark());
  }

  private apply(dark: boolean): void {
    this.isDark.set(dark);
    document.documentElement.setAttribute(
      'data-theme', dark ? 'dark' : 'light'
    );
    localStorage.setItem(
      this.STORAGE_KEY, dark ? 'dark' : 'light'
    );
  }
}

// app.component.ts — call on startup
export class AppComponent implements OnInit {
  constructor(private theme: ThemeService) {}
  ngOnInit() { this.theme.init(); }
}


// ── Flutter: ThemeMode state via Riverpod ─────────
// lib/providers/theme_provider.dart

import 'package:flutter_riverpod/flutter_riverpod.dart';
import 'package:shared_preferences/shared_preferences.dart';

final themeModeProvider =
  StateNotifierProvider<ThemeModeNotifier, ThemeMode>(
    (ref) => ThemeModeNotifier()
  );

class ThemeModeNotifier extends StateNotifier<ThemeMode> {
  ThemeModeNotifier() : super(ThemeMode.system) { _load(); }

  Future<void> _load() async {
    final prefs = await SharedPreferences.getInstance();
    final saved = prefs.getString('themeMode');
    if (saved == 'dark')  state = ThemeMode.dark;
    if (saved == 'light') state = ThemeMode.light;
  }

  Future<void> toggle() async {
    final prefs = await SharedPreferences.getInstance();
    state = state == ThemeMode.dark
      ? ThemeMode.light : ThemeMode.dark;
    prefs.setString('themeMode',
      state == ThemeMode.dark ? 'dark' : 'light');
  }
}
